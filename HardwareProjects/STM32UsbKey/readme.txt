
В системе устройство определяется как HID - совместимое устройство и USB - устройство ввода.
.\Firmware\src\usb_prop.c - содержит логику транспортного уровня.
.\Firmware\src\usb_endp.c - содержит логику транспортного уровня.
.\Firmware\src\hw_config.c - настройки периферии. Порты ввода/вывода, DMA, ADC, тактирование и т.д.
   Вывод к которому подключен синий светодиод настроен в Set_System();

.\Firmware\inc\commands.h - идентификаторы команд.
.\Firmware\inc\pins.h - описание настроек для кнопки.

Для работы с платой используется ПО для ПК находящееся в .\Native\HardwareKeyExample.
Помигать светодиодом и считать состояние кнопки (подключается к PB9, замыкается на минус) можно 
раскомментировав строки в проекте .\Native\HardwareKeyExample\KeyProger\ , методе KeyProger::TimerTick().
ПК будет передавать команды на задание состояния выходу и читать состояние входа.
При желании количество вводов/выводов можно увеличить. Для этого в коде прошивке МК 
нужно инициализировать соответствующие выводы. В main() прописать получения данных с новых выводов.
